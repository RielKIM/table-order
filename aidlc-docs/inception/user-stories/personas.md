# User Personas

## Persona 1: 고객 (Customer) — "김민수"

| 항목 | 내용 |
|---|---|
| **이름** | 김민수 (대표 고객) |
| **나이** | 25~55세 (폭넓은 연령층) |
| **역할** | 매장 방문 고객 (테이블 이용자) |
| **기술 수준** | 스마트폰/태블릿 기본 조작 가능 |
| **사용 환경** | 매장 내 테이블에 설치된 태블릿 (브라우저 기반) |

### 목표 (Goals)
- 빠르고 간편하게 메뉴를 확인하고 주문하고 싶다
- 주문한 음식의 현재 상태를 알고 싶다
- 추가 주문을 쉽게 하고 싶다
- 직원을 부르지 않고도 주문 진행 상황을 확인하고 싶다

### 니즈 (Needs)
- 직관적인 메뉴 탐색 (카테고리별 분류, 시각적 표현)
- 별도 로그인 없이 즉시 사용 가능
- 장바구니에서 쉽게 수량 조절 및 삭제
- 주문 후 명확한 확인 피드백
- 현재 세션의 주문 이력 한눈에 파악

### 불만 요인 (Pain Points)
- 복잡한 UI, 너무 많은 단계
- 느린 반응 속도
- 주문 상태를 알 수 없어 불안감
- 새로고침 시 장바구니 초기화

### 특성 (Characteristics)
- 태블릿 터치 인터페이스에 익숙
- 인내심이 제한적 (3초 이상 대기 시 이탈 경향)
- 시각적 정보(이미지, 가격)에 의존
- 여러 번 추가 주문 가능

## Persona 2: 관리자 (Admin) — "박정희"

| 항목 | 내용 |
|---|---|
| **이름** | 박정희 (매장 운영자) |
| **나이** | 30~60세 |
| **역할** | 매장 사장/매니저 (운영 관리자) |
| **기술 수준** | PC/태블릿 중급 사용자 |
| **사용 환경** | 매장 카운터 PC 또는 태블릿 (브라우저 기반) |

### 목표 (Goals)
- 실시간으로 들어오는 주문을 빠르게 확인하고 처리하고 싶다
- 테이블별 현재 상태를 한눈에 파악하고 싶다
- 메뉴를 유연하게 관리하고 싶다
- 매장 운영을 효율적으로 하고 싶다

### 니즈 (Needs)
- 신규 주문 즉시 알림 (시각적 강조)
- 테이블별 주문 현황 대시보드
- 주문 상태 빠른 변경 (원클릭)
- 잘못된 주문 삭제 기능
- 테이블 세션(이용 완료) 관리
- 과거 주문 내역 조회
- 메뉴 등록/수정/삭제

### 불만 요인 (Pain Points)
- 주문이 누락되거나 지연되는 것
- 여러 테이블 주문을 동시에 관리하기 어려운 것
- 메뉴 변경이 즉시 반영되지 않는 것
- 영업 중 로그인이 풀리는 것 (16시간 세션으로 해결)

### 특성 (Characteristics)
- 바쁜 영업 시간에 빠른 조작 필요
- 여러 테이블을 동시에 모니터링
- 메뉴 변경은 주로 영업 외 시간에 수행
- 일일 매출/주문 현황 확인 필요 (과거 내역)
